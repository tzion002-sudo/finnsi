# המצפן V3.0 — עיצוב ייצוב הסנכרון (שלבים 0-1)

**תאריך:** 2026-07-06 · **סטטוס:** שלבים 0-1 מומשו ואומתו · שלבים 2-3 ממתינים לשבוע ייצוב

## הבעיה

המערכת פספסה עדכוני דיבידנדים שוב ושוב (למשל הדיבידנד של 25.6.2026). שורש הבעיה **לא** היה בסקנר —
GitHub Actions רץ אמין כל בוקר — אלא ב"קילומטר האחרון":

1. הסקנר פרסם רק דיבידנד אחד (`nextDividend`) ל-`market_data/latest`.
2. ה-Dashboard הוסיף אותו לרשימה הקבועה **רק בלחיצת "החל"** במודל הבוקר.
3. ה-backfill המלא הוסר ב-V2.9.1 בגלל באג כפילויות אחרי מחיקה ידנית.
4. קליק שפוספס בחלון של ~שבוע = דיבידנד שאבד לתמיד.

בזמן המימוש נמצאו בפועל **7 דיבידנדים שאבדו** כך לאורך השנה
(2025-07-03, 2025-07-31, 2025-08-28, 2025-10-16, 2025-10-23, 2026-01-29, 2026-06-25) — כולם שוחזרו אוטומטית.

## הארכיטקטורה (הוחלטה במועצת קלוד — 5 יועצים + ביקורת עמיתים)

### דיבידנדים server-authoritative
- **הסקנר** ([scripts/daily-scanner.js](../../../scripts/daily-scanner.js), שלב 3ב) כותב את הרשימה
  המלאה ל-`market_data/msty_dividends` עם ולידציית-סבירות:
  - amount בטווח (0, 5); exDate בפורמט ISO ולא עתידי מ-90 יום
  - הגנת הצטמקות: רשימה חדשה קטנה מ-50% מהקיימת → לא נכתבת + warning
  - נכתב רק ממקור חי (yieldmax/yahoo) — לא מ-cache
- **מודול טהור** [src/lib/dividendSync.js](../../../src/lib/dividendSync.js) — `reconcileDividends(server, local, tombstones)`:
  - התאמה גמישה: exDate מדויק, או |Δ| ≤ 2 ימים עם אותו סכום (תיקוני תאריך רטרואקטיביים)
  - מחיקה ידנית = tombstone ב-`settings.deletedDividends` — לא קמה לתחייה
  - עריכה ידנית (`verified: true`) מנצחת את השרת
  - רצפת תאריך (הדיבידנד הלוקאלי המוקדם / 2025-06-01) — לא מציף היסטוריה מלפני ההחזקה
  - אידמפוטנטי
- **Dashboard** — subscription ל-`msty_dividends` + reconcile שקט. אין תלות במודל/קליק.
  לקח קריטי: חובה `cloudUpdateRef.current = false` לפני `setMstyDividends(merged)`,
  אחרת ה-debounce מסווג את השינוי כ-echo מהענן ולא שומר.
